# Lessons

- If a user screenshot shows UI text that no longer exists in source, treat it as a runtime artifact mismatch first: rebuild + redeploy and verify source/runtime DLL timestamps before further code changes.
- When users need to inspect model space during PLSR Accept/Ignore, keep the review UI modeless and drive apply/cancel via explicit button flags; relying on modal DialogResult blocks pan/zoom and feels like command lockup.
- For UI-driven `Editor.GetEntity` prompts inside ATSBUILD flows, always normalize the command prompt in a `finally` (`WriteMessage` + `PostCommandPrompt`) so stale boundary-pick prompts do not linger on the command bar after returning to the window.
- For UI-initiated editor selection (for example boundary pick from ATSBUILD window), avoid hide/show round-trips; use `Editor.StartUserInteraction(windowHandle)` so AutoCAD command context and prompt lifecycle are released cleanly.
- In ATSBUILD command flows, keep PLSR Accept/Ignore review modal by default; modeless review can leave command-context prompts lingering and invalidate apply state unless command ownership is re-architected.
- In PLSR flows, avoid redundant modal summaries after `PLSR Review`; keep one decision UI and write detailed results to log/file instead of stacking extra popups.
- If users rely on skip diagnostics for reconciliation, do not truncate skipped-item examples; provide the full skipped list (or explicit export) so counts and details align.
- If PLSR missing-label fallback can only create owner/disp text without real source geometry, do not create it (even behind env toggles); skip creation and show a warning dialog with examples instead of producing floating `MText`.
- If users report missing disposition labels with many `Skipped (not closed)`, treat trimmed/open OD polylines as recoverable candidates: attempt bounded endpoint-gap closure + area validation before excluding them.
- When PLSR summary shows high `Missing labels` but low `Actionable results accepted`, keep no-source/no-template rows as explicit skip+warning outcomes; do not create XML/text-only fallback labels.
- If PLSR missing-label rows remain non-actionable due absent quarter-matching source geometry, add a safe same-DISP template fallback (existing label -> new quarter placement) instead of leaving rows as permanent skips.
- When a UI toggle must stay off by default, do not bind its default to persisted `Config` values in shared option catalogs; use explicit unchecked defaults and only override from intentional seed-state recovery.
- In boundary round-trip modal flows, `section_requests_empty` snapshot failures should trigger a bounded reopen (without seed) instead of immediate cancel; otherwise valid surface-impact runs can abort before explicit Build is captured.
- For quarter-dependent workflows, decouple internal quarter-geometry processing from user-visible quarter-definition output: allow processing to run for reliability, then conditionally clean up `L-QSEC`/`L-QUATER` when the `1/4 Definitions` toggle is off.
- In modal UI auto-close recovery paths, never force only one related toggle (for example `Check PLSR`) when multiple dependent options exist; persist and restore the full option set (`Check PLSR` + `Surface Impact`) so recovery behavior matches user intent.
- For UI wording/layout requests, if the user references a recent screenshot, inspect local reference assets first (for example `REFERENCE ONLY`) and align labels/order to that source before asking for additional Figma links.
- Correction-line context cannot rely on plain ATS grid adjacency at township seams; force include shifted cross-range neighbors for seam-edge sections (for example `6 above -> 35 below+range`, and mirror the rule below the seam).
- When users validate one side of a correction seam as fixed, explicitly test/cover the opposite seam direction too; missing symmetric pairs (for example `32 below -> 5 above`) can still drop required RA context.
- Forced correction-context sections can be valid for RA generation but noisy for correction-line/LSD seam inference; keep those scopes decoupled to avoid unintended LSD endpoint behavior.
- For midpoint snapping, prioritize exact regular boundary midpoints first when an LSD endpoint is already on `L-SEC`/`L-USEC`; quarter/component midpoint logic should be fallback, not primary.
- When repeated endpoint heuristics regress nearby correction logic, pivot to an explicit rule-matrix pass keyed by section/quarter instead of layering more fallbacks into the legacy path.
- Treat generic `L-USEC` in user LSD rules as blind-line layer (`L-USEC`) only, not `L-USEC-3018`; if the user says endpoints must never end on 3018, encode that as a hard matrix constraint.
- For Rule 1 (1/4 inner endpoints), derive targets from final adjusted `L-QSEC` components in-model; section-anchor-derived centers/endpoints can be stale after endpoint cleanup and produce exact half-gap misses (for example 5.03 m on west halves).
- When deriving post-adjustment `L-QSEC` component targets, scope candidate components to the owning section envelope; global clip-window clustering can bleed into adjacent sections and pull a 1/4 endpoint onto the neighbor section line.
- In deferred LSD redraw, never erase by "segment intersects requested window" alone; boundary-touching LSDs from adjoining previously-built sections can be deleted. Use quarter ownership (midpoint-in-requested-quarter) for erase scope.
- For horizontal LSD outer endpoints, honor surveyed `L-SEC` as first-choice target; if `L-USEC` is prioritized first, lines can cross the section line into the adjoining side.
- Even with `SEC` priority, midpoint target search must be section-scoped; otherwise global candidate pools can select neighboring-section `L-SEC` midpoints and drive E-W LSDs across the intended boundary.
- Correction seam classification must not rely on north-side evidence only; when north RA is out-of-buffer, use surveyed south-edge horizontal evidence before classifying seam as unsurveyed (`L-USEC-C`).
- For correction-line bugs in 100m-buffer/range-over builds, add explicit seam-input logging (north/south sample contributions, one-sided synthesized seam mode, and xOnly vs seam-band evidence counts) before changing thresholds.
- One-sided synthesized correction seams (missing north/south section samples in 100m-buffer runs) need relaxed surveyed-edge tolerance and x-only fallback checks; strict seam-band edge tolerances can miss valid surveyed south-only evidence and incorrectly classify seams as `L-USEC-C`.
- If ATSBUILD ends mid-run with no exit marker and the last line is `Starting shapefile import.`, treat it as a native importer crash boundary; add step logs around `Importer.Init`/`Importer.Import` and refuse unsafe large-survey imports when location-window setup fails.
- Never call native Map importer on an unvalidated `.shp`: pre-check `.shp/.shx/.dbf` integrity first, and if the direct file is corrupt, resolve the newest valid matching copy before `Importer.Init` to avoid hard process crashes.
- Shape auto-update should copy only configured shape sets (same basename sidecars), not recursive folder contents; recursive selection is allowed only to locate the best valid source set, and destination copy stays flat/specific.
- For quarter-line endpoint snaps near correction seams, strict line-vs-segment intersection is not sufficient; include a bounded apparent-intersection fallback against short segment extensions so 1/4 endpoints can still land on the intended projected hard-boundary intersection.
- For quarter-view 1/4 geometry at blind-line RA crossings, do not rely on endpoint-style anchor intersections; preserve perpendicular crossing by projecting the south correction segment at center-U, then use anchor intersection only as fallback.
- When a user asks to verify an exact landing coordinate, add explicit runtime verification logs with a non-suppressed prefix and do not treat the fix as complete until those logged coordinates are checked against the provided target.
- If quarter south-mid still drifts after a projection fix, verify that correction-south classification is actually active; add a promotion path from regular south boundary to best-fit `L-USEC-C-0` candidate and avoid selecting the absolute south-most correction segment when its road-allowance offset fit is worse.
- For skewed quarter geometry, constant-U projection can miss the intended blind-line corner by a few meters; prioritize apparent intersection against the actual quarter divider (`BottomAnchor -> TopAnchor`) with bounded segment extension, then fall back to projection.
- If the 1/4 south-mid corner still misses by a few meters after pre-clamp fixes, check the post-draw quarter-box vertex snap stage; it can re-drift the computed corner, so protect that shared SW/SE south-mid vertex from generic snap replacement.
- For "extend the 1/4 line to south RA boundary" cases, apply a final south-mid authority pass using the active divider trajectory (`center -> north divider`) intersected with the south boundary segment; this catches residual drift even after pre-clamp fixes and snap guards.
- When users define the target as extending the actual 1/4 line, use the real `L-QSEC` divider segment as the intersection reference; anchor-derived divider proxies can be offset enough to miss the required apparent crossing.
- If divider-extension logs show `found=False` with very large required extension, the south boundary selector likely picked a non-spanning fragment (often east-only). Penalize segments by divider/mid-U coverage gap so the selected south segment actually spans the 1/4 divider crossing.
- For SW quarter SW-corner misses on correction/surveyed cases, resolve the corner from infinite west-boundary x infinite south-boundary intersection (apparent crossing) and avoid re-clamping that resolved point back to fallback south limits.
- If NE/SE quarter corners are built from `frame.EastEdgeU`, sloped east RA boundaries will miss apparent intersections; resolve east boundary from segment geometry and compute `east x south` / `east x north` corner intersections directly.
- When a user reports a "1/4 definition" miss, verify the fix path targets quarter definition objects (`L-QUATER` / `L-QSEC`) and not LSD subdivision-only enforcement paths before accepting the change.
- In LSD outer-endpoint rule-matrix selection, scoring by endpoint move can skip the first valid boundary and cross to a farther one; prefer nearest outward boundary from the inner anchor and preserve endpoints already on preferred boundaries.
- For west-side 1/4 definition points, apply apparent `west x north` intersection logic (not only `west x south`) and protect computed west vertices from generic post-draw corner snapping; otherwise non-correction `NW/W.1/2/SW` points can drift.
- When `VERIFY-QTR-SOUTHMID-DIVEXT-INPUT` shows `found=False` and the selected south segment U-range does not contain the active divider U, treat boundary selection as wrong even if endpoints look plausible; rank south/north candidates with a strong divider-span penalty.
- For blind non-correction 1/4 definitions, divider-span penalty alone can still pick impossible boundary fragments; enforce a hard candidate gate that requires bounded apparent intersection with the active divider (south) and prefer west-linkable north candidates when west-side points are the acceptance criteria.
- In blind non-correction sections, west boundary can be split across north/south halves; do not drive both `S.W.1/4 S.W.` and `N.W.1/4 N.W.` from a single west segment intersection. Add explicit west-south endpoint authority plus west-band hard-corner snap for north corners.
- When a corner is intentionally locked by apparent-intersection/endpoint authority, do not re-clamp its U/V afterward; late clamps can silently reintroduce meter-scale drift.
- For small residual SW-corner drift after apparent west/south intersection (sub-2m scale), add a final high-confidence hard-corner snap (priority 0, tight move cap) rather than retuning global boundary selectors.
- In skewed quarter frames, dot-product world->local conversion can produce meter-scale drift at north/south extremes; use exact basis inversion for authority points (divider endpoints, hard-corner snaps, and west endpoint locks) before writing local coordinates.
- For blind NW corner snap, seeding hard-corner ranking with center-west U can bias to east wrong candidates; seed snap from raw apparent `west x north` intersection instead.
- When a raw apparent `NW` intersection exists, treat it as authority and allow only micro-snap refinement; large blind-snap moves can jump to unrelated north-band corners.
- In skewed quarter frames, any helper that intersects world boundary lines in local space must use exact basis inversion conversion; dot-product local projection inside generic intersection helpers can introduce 20m+ apparent-corner drift even when world-line geometry is correct.
- In blind NW raw-lock logic, a tight negative north-offset gate can reject valid apparent intersections (raw exists but lock does not), causing fallback to unrelated corners; blind sections need a wider negative north-offset acceptance band before snap/fallback.
- Blind SW endpoint authority (`SW-SW-END`) must not apply from midpoint-only west segments; require endpoint proximity to south boundary before override or `sw_sw` can collapse onto `w_half`, creating 3-point quarter definitions.
- Post-draw quarter vertex snap must protect any corner already locked by apparent-intersection authority on east side (`NE.NE`/`SE.SE`) as well as west/south protected corners; otherwise a single hard-corner snap can re-drift a correct quarter-definition corner.
- East-corner protection should not depend only on apparent-lock flags; strict-intersection fallback paths can still yield correct `NE.NE`/`SE.SE` points and must be protected from post-draw snap whenever east boundary + north/south boundary geometry is present.
- For blind sections, selecting east boundary solely by midline proximity to `EastEdgeU` can choose the wrong vertical segment for `NE.NE`; add a north/south-constrained east-segment selector that scores by apparent intersection offsets against both resolved north and south boundaries.
- If an east selector silently returns no candidate, add explicit `found=False` diagnostics before retuning geometry; otherwise repeated fixes can appear as \"no change\" because the new path never engaged.
- For blind non-correction `N.E.1/4` corners, treat the interior hard corner (south-running x west-running `L-SEC/L-USEC-0` node) as final authority; projected east/north intersections can miss that node by large offsets.
- Blind `N.E.1/4` hard-corner snap must reject north/east exterior candidates: enforce positive interior insets (east/north), prefer RA-width inset fit, and block snaps that move farther north/east than raw apparent intersection.
- Stale/overlap cleanup must not erase adjacent 1/4 geometry that only shares a single endpoint with the requested build: require full section ownership for `L-QUATER` erase and skip shortest-wins deletes for single-endpoint touches.
- For `N.E.1/4` on blind sections, cluster-only snapping is not sufficient: solve the corner from the selected east segment intersecting a west-running boundary segment that actually reaches that node (bounded extension), and only use cluster snap as fallback.
- Enforce endpoint authority by section type: non-correction-adjoining sections must resolve `N.E.1/4` at real endpoint nodes (horizontal+vertical endpoint evidence), while endpointless apparent intersections are allowed only on correction-adjoining sections.
- After introducing section-type endpoint authority, prune dead/unsafe fallback paths: non-correction NE branches should not continue into hard-node/cluster fallbacks, or they can reintroduce endpoint misses.
- When post-correction LSD endpoint enforcement runs, never pass combined requested+context quarter infos into the rule-matrix; scope quarter contexts to requested ids only or adjacent prebuilt sections can be mutated by later builds.
- `Check PLSR` must not be gated by `Disposition labels`: always audit existing disposition labels first, reuse same-quarter labels by `dispNum`, and only generate missing labels when label generation is explicitly enabled.
- For user-facing bulk data checks (like PLSR), provide an explicit review gate with per-row `Accept/Ignore` and `Accept All/Ignore All`; only actionable rows should be applied, and non-actionable findings should remain informational.
- For PLSR label matching, never assume the disposition number is the last text line; labels can include trailing status lines like `(Expired)`. Extract disp numbers by prefix+digits from all lines (bottom-up) to avoid false missing-label reports.
- AutoCAD `MText.Contents` may include inline formatting controls (for example `\A1;`); always strip control codes before owner/disp parsing and normalization or PLSR checks will show false mismatches/missing labels.
- In PLSR review UI, distinguish quarter-level findings from disposition-level findings explicitly (for example `Disp=N/A` on `Missing quarter in XML`) so blank disp cells are not mistaken for parsing bugs.
- PLSR review should never display raw `MText.Contents`; flatten sanitized lines for UI output, because raw content carries control groups (`\P`, `\pxqc;`, etc.) that users interpret as parsing failures.
- For PLSR expiry checks, do not create `Add (Expired)` actions when label text already contains an expired marker; detection must be normalized/sanitized (not literal raw `(Expired)` string matching only).
- When copy-to-`build` fails (locked DLL), users may unknowingly run stale plugin code; always verify `bin` vs `build` timestamps/sizes and sync artifacts before asking for retest.
- In review grids, never gate `Accept/Ignore` editing by actionability at cell-edit time; users expect dropdowns to work for all rows. Keep actionability enforcement in apply logic, and force commit of in-cell edits before closing dialogs.
- `Check PLSR` runs must be able to generate missing labels independently of the `Disposition labels` toggle: scan existing OD disposition polylines in requested scope first, then import shapefiles only as fallback when none are present.
- If A-DIM is the required standard, remove style toggles from UI and force aligned-dimension placement in code paths; keeping optional leader toggles invites mixed output styles.
- For PLSR workflows, missing-label generation must be review-driven: do not pre-place all labels when `Check PLSR` is on; surface `Missing label` as actionable and create only accepted rows.
- In mixed-action PLSR apply loops, never gate all actionable rows on `issue.Label != null`; `CreateMissingLabel` actions require `Disposition+Quarter` and must be routed by `ChangeType` so accepted missing-label rows are actually created.
- For `Check PLSR` performance, never force fallback disposition shapefile import before proving missing labels exist; precheck XML vs existing labels first, then import only when missing-label creation is actually needed.
- For `/debug-config` performance, high-volume trace logging must be buffered and selectively suppressed by default; keep deep traces (`TRACE-LSD-*`, per-segment relayer logs, wellsite debug spam) opt-in via env toggles.
- When adding pre-build UI actions that depend on derived geometry (for example quarter definitions), do not assume helper layers (like `L-QUATER`) already exist in drawing state; derive from authoritative section-index inputs in-memory and avoid leaving temporary helper entities visible.
- For PLSR runtime paths, guard AutoCAD DB reads/writes per-entity/per-issue and keep a top-level stage-aware catch in `RunPlsrCheck(...)`; uncaught `Autodesk.AutoCAD.Runtime.Exception` in scan/apply can terminate ATSBUILD like a fatal.
- Boundary-import UI actions should remove placeholder blank grid rows before appending imported entries; leaving a blank first row is usually harmless but creates carry-down ambiguity and hides real failure patterns.
- For native-crash-prone paths (PLSR precheck + shapefile importer), flush boundary log lines immediately (`PLSR precheck`, `Starting shapefile import`, `Importer.Init/Import begin`) so crash location survives abrupt termination.
- For boundary-expanded PLSR runs, any candidate-level geometry test (`IsPointInsidePolyline`, overlap probes, `GeometricExtents`) must be wrapped and skipped per candidate; never allow one malformed polyline to take down the full check.
- Exception code `0xE0434352` indicates a managed exception escaped to host; always add top-level stage-aware catch/logging in command handlers and per-event UI try/catch in WPF/WinForms build submit paths to prevent hard host termination.
- WPF `Window.DialogResult` can only be set for modal dialogs (`ShowDialog`); shared window handlers used by debug tools must use a guarded close helper that falls back to `Close()` for modeless launches.
- After making modal-safe close changes, do not keep ATSBUILD gated on `ShowDialog()==true`; accept a populated UI result payload (`window.Result != null`) as the execution signal or valid builds will be logged as `ui_cancelled`.
- Do not treat generic UI auto-close (`DialogResult != true` with no build intent flags) as permission to run ATSBUILD; fallback recovery must require explicit build intent (`BuildRequested` or `BuildAttempted`) unless there is an explicit opt-in auto-run mode.
- In WPF modal flows, `Hide()` used for in-drawing boundary selection can produce a false dialog return before Build; handle this as a boundary round-trip resume case (reopen dialog with seeded state), not as cancel and not as implicit auto-run.
- If UI closes with `ExplicitCancel=False` and no build-intent flags, prefer reopen-with-seeded-state (explicit Build still required) over immediate cancel; cap reopen retries and gate snapshot-run fallback to non-aborted build traces to avoid both self-cancel and unintended autorun.
- For modal/no-result UI recovery, always instrument section payload flow (`grid rows -> parsed requests -> snapshot -> final input`) before changing fallback logic again; otherwise duplicate-window artifacts can hide whether the real fault is lifecycle or dropped section state.
- Once behavior is confirmed stable by the user, remove temporary high-volume diagnostics and keep only durable guardrails; leave enough logs for state transitions, not full payload dumps.
- Persisted runtime options used for backend recovery (for example PLSR fallback options) should not also be applied as constructor-time UI defaults; boundary/no-seed reopen flows can unintentionally turn user-facing toggles ON.
- In PLSR fallback mode, never feed scanned existing disposition entity IDs into cleanup erase paths; keep imported IDs separate and only erase imported disposition linework when linework output is off.
- In PLSR-only runs (`Check PLSR` on, disposition linework/labels off), do not gate fallback disposition import on `existingDispositionCount == 0`; partial existing coverage can still leave many missing labels without source geometry, so run missing-label precheck and allow supplemental import when gaps exist.
- If a PLSR supplemental import path reaches `Importer.Import begin.` and AutoCAD hard-terminates, default back to non-import behavior for partial existing-coverage runs and require explicit env opt-in (`ATSBUILD_PLSR_SUPPLEMENT_IMPORT=1`) before re-entering the native importer path.
- When `/debug-config` still creates far fewer labels than missing count, check whether `Actionable results accepted` is low due strict disp-number normalization; normalize DISP_NUM canonically (strip non-alnum, normalize prefix+numeric suffix, trim numeric leading zeros) so existing/source matching and per-quarter dedupe use the same canonical key.
- If ATS Fabric (sections) is enabled, keep 1/4 linework visible even when the UI `1/4 Definitions` toggle is off; cleanup should only erase quarter view/helper lines when ATS Fabric, 1/4 Definitions, and env quarter-view override are all off.
- When ATS Fabric must preserve quarter helper behavior but `1/4 Definitions` is off, split cleanup gates by layer role: `L-QUATER` should follow the explicit 1/4 display toggle/env only, while ATS-specific helper-line retention can remain independent.
- For multi-toggle geometry visibility behaviors, centralize decision logic in one pure policy helper and consume it from both planning and cleanup paths to prevent matrix drift bugs.
- For complex cleanup behavior, separate decision computation from AutoCAD side effects with a pure cleanup plan object; this keeps toggle matrices testable and prevents regressions when new visibility rules are added.
- If ATSBUILD logs stop at `Importer.Import begin.` with no exit marker, treat it as a native Map importer hard-crash boundary and add pre-import risk guards (size/unsafe conditions) with explicit env override instead of invoking native import unconditionally.
- For very large disposition shapefiles (`DAB_APPL`-scale), do not call native `Importer.Import()` on full source files; build a temporary spatially filtered SHP/SHX/DBF subset first and import that subset to avoid AutoCAD host termination.
- If raw spatial subset filtering returns zero matches on large shapefiles, do not skip import outright; fall back to chunked safe import because CRS/projection differences can make raw coordinate filtering miss valid in-scope records.
- For ATS-fabric-off large imports, treat per-chunk MPOLYGON volume as a crash risk; keep default chunk sizes conservative (smaller) and only scale up via env override after stability is proven.
- Do not rely on default `scopeBufferMeters` for ATS disposition imports; pass the intended buffer explicitly (`100m`) so runtime scope and log output match user expectations.
- If large-file spatial subset keeps returning zero matches while native import still finds features, verify SHP CRS first; when source is geographic and drawing is UTM, transform section extents to source coordinates (zone-aware) before record-bounds filtering, otherwise performance collapses into full chunk fallback.
- If a new import acceleration path is unstable in one execution mode (for example ATS Fabric OFF), gate the optimization by mode and keep the proven stable fallback for that mode instead of forcing one path for all builds.
- If logs still stop at `Importer.Import begin.` while importing a generated spatial-subset shapefile, treat subset-file import itself as unstable and default large-file runs to chunked source import; keep subset-file import behind explicit env opt-in only.
- When a user asks for validation linework only as a check, keep it behind an explicit UI toggle and avoid forcing always-on drawing behavior.
- For folder-level photo QA workflows, expose them as explicit palette actions with simple picker prompts (buffer + folder) rather than overloading the main build command flow.
- For zone-specific buffer modes (for example `PROPOSED / 100m`), prompt for each zone's boundary and block explicitly, and use deterministic zone precedence to prevent duplicate inserts in overlapping areas.
- When WLS location rows must respect ATS road-allowance quarter geometry, resolve findings against existing `L-QUATER`/`L-QUARTER` polygons first and only fall back to synthetic section-index quarter splits if no layer polygon contains the point.
- WLS table location quarter tokens should be rendered as dotted legal abbreviations (`N.W.`, `N.E.`, `S.W.`, `S.E.`), not compact `NW/NE/SW/SE`.
- When WLS users request legal-location detail beyond quarter direction, use ATS-compatible LSD numbering (`4x4` serpentine per section) and output `LSD-SEC-TWP-RGE-WMER` rather than quarter tokens.
- For WLS `PROPOSED / 100m` table outputs, enforce explicit group ordering (`PROPOSED` then `100m`) and render a visible blank separator row between groups to match review workflow expectations.
- Photo caption styling requests may target only part of the string (`PHOTO #` vs number); use MText inline color formatting so label text color can differ from numeric token color without affecting the finding-description line.
- For WLS quarter-boundary resolution, treat ATS `L-QUATER` as authoritative quarter geometry (road-allowance-aware) and only fall back to `L-QUARTER` when `L-QUATER` is absent.
- If quarter validation linework is meant to verify ATS quarter usage, do not mix in synthetic section-index quarter polygons; validation output should reflect matched quarter-layer polygons only.
- When users ask for photo caption style changes, apply both text transform and placement offset explicitly (`ALL CAPS`, color, and exact Y shift) rather than partial formatting tweaks.
- For WLS grouped table spacer rows (for example between `PROPOSED` and `100m`), treat them as visual separators: side/vertical borders off, top+bottom borders on, and explicit row height so print layout is predictable.
- For WLS ATS quarter matching, do not hardcode one quarter layer source before findings are known; load all candidate quarter layers and select the source that maximizes containment on active findings, with ATS `L-QUATER` as tie-break preference.
- If WLS L-QUARTER validation linework is enabled, do not require strict containment-only matches to draw diagnostics; add nearest-quarter fallback drawing and explicit source-selection fallback so users always get visible validation geometry when quarter layers exist.
- For WLS ATS quarter-boundary fidelity, if the user requests ATS-only behavior, enforce `L-QUATER`-only source loading (no `L-QUARTER` fallback) so road-allowance quarter logic always comes from ATS-built quarter view geometry.
- If the user says validation must use/draw `L-QUARTER` (not `L-QUATER`), make quarter-source loading include and prioritize `L-QUARTER`; do not hard-lock to ATS `L-QUATER` only.
- Keep a single canonical quarter layer name in WLS (`L-QUARTER`); remove legacy `L-QUATER` aliases from matching logic to avoid ambiguous source selection and user confusion.
- For WLS L-QUARTER validation drawing, do not depend solely on existing quarter layer polylines; when sources are absent, populate validation polygons from `AtsQuarterLocationResolver` quarter vertices so ATS road-allowance logic still draws for user verification.
- When WLS must match ATS quarter-definition behavior (road allowances, 20.12-offset workflow), use ATS section-build internals to generate quarter definitions and immediately run ATS cleanup to strip temporary section linework, rather than relying on simplified in-house frame splitting.
- For quarter/LSD boundary resolution regressions, do not let `L-USEC-0` silently win when 20.12-class candidates exist on requested west/south sides; add an explicit preferred-layer promotion pass (with zero-offset fallback) after initial selection.
- For LSD endpoint enforcement, do not assume `onThirty` endpoints can always resolve within strict midpoint/axis thresholds; keep a widened 30.18-near tolerance and a relaxed hard-boundary escape fallback so `L-SECTION-LSD` endpoints cannot remain on `L-USEC-3018`.
- In LSD endpoint enforcement boundary collection, do not require boundary polylines to be collinear/open-only; collect per-segment geometry (including closed polylines) for `L-USEC*`/`L-SEC*` layer scans so 30.18 targets are never missed.
- When adding LSD endpoint diagnostics, instrument both execution paths (TryEnforceLsdLineEndpointsByRuleMatrix and fallback EnforceLsdLineEndpointsOnHardSectionBoundaries); the rule-matrix path can short-circuit and otherwise hide all new traces.
- If LSD rule-matrix logs show repeated outer source=fallback-anchor moved=False and outerAdjusted=0, check boundary midpoint scope gating first; sectionScopePad must accommodate RA-offset boundaries (~20-30m) or TWENTY/ZERO targets are skipped.
- In LSD rule-matrix, do not short-circuit horizontal outer endpoints directly to quarter anchors; route them through preferred boundary midpoint selection (TWENTY/ZERO/SEC) first, with anchor only as fallback, or west/east endpoint fixes will never engage.
- In LSD rule-matrix midpoint targeting, do not always preserve current endpoint on first-priority BLIND segments; parallel pre/post-adjust blind rows can coexist. Require strong hard-link evidence (both endpoints near SEC/0/20/CORRZERO) before freezing BLIND endpoints, otherwise continue scoring.
- In LSD midpoint selector, never return immediately on 'already on primary boundary' checks; mark as preservable and run midpoint scoring first, then preserve only if no valid midpoint candidate exists. Immediate preserve can lock onto stale pre-adjust ZERO/BLIND rows.
- For WLS quarter-location behavior, distinguish clearly between the in-DLL section-index resolver (simple frame split) and the ATS reflection fallback that runs `DrawSectionsFromRequests`; full road-allowance quarter generation requires `AtsBackgroundBuilder.dll` to be loadable.
- For non-correction `N.E. N.E.` quarter corners, never fall back blindly to the highest east-boundary endpoint; require north/horizontal endpoint-node evidence (or near-north-boundary touch), otherwise keep the prior NE candidate to avoid crossing RA.
- For quarter-definition ownership baselines, target full `RoadAllowanceUsecWidthMeters` (20.12-class) for west/south/east side selection; using `RoadAllowanceUsecWidthMeters - RoadAllowanceSecWidthMeters` (10.05-class) under-allocates quarter ownership and can pin the east side on the east edge instead of inward (west).
- In mixed surveyed/unsurveyed quarter boundaries, do not allow west-side preferred-layer promotion or fallback ladders to downgrade from 20.12 ownership to 10.05/0 when south is surveyed; keep downgrade paths only for blind-south cases.
- Apply the same ownership rule to south: for surveyed south boundaries, do not run fallback ladders that can downgrade full 20.12 ownership to SEC/near-edge candidates.
- For quarter-definition ownership, use the full SEC-width inset (`RoadAllowanceSecWidthMeters` ~= 20.11), not `RoadAllowanceUsecWidthMeters` (~30.16); switching to USEC-width over-shifts south/west/east by about 10.05m.
- Quarter ownership inset is side-dependent in mixed surveyed/unsurveyed contexts: south/east can be SEC-width while west may still require USEC-width; avoid applying one global inset constant to all sides.
